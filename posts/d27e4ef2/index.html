简历编写

Posted on 2021-08-13 | In 678

优先简历具备的特点

对于程序员行业来说，一份优秀的简历，需要具备几个特点：

  1. 让招聘者看了很舒服，不会看了就想丢垃圾桶里。
  2. 让招聘者能很快看到自己想要看到的东西。

招聘者可能每天都需要看上百份、甚至上千份简历，一份简历的查看时间可能是按秒计算，没那么多时间去从一堆垃圾信息中寻找亮点。

总结下来就两个词：简洁工整、突出重点。

简洁工整

  1. 简历整体排版上要工整、结构清晰，字体尽量保持一致。
  2. 页数不超过2页，经验较少的1页，较多的2页，超过了，说明你肯定说废话了。
  3. 写完检查几遍，不要有错别字，这个会让招聘者觉得这个人不细心或者不重视。
  4. 白底黑字，不要有太多花里胡哨的颜色，在标题部分可以有一些色彩，但是不要太多，不要太刺眼。
  5. 保存时使用PDF，这个细节很多人不知道，WORD 文档在不同设备上的格式排版可能会乱掉，导致根本没法看，所以记得一定要用PDF。

突出重点

  1. 关键的信息可以放在显眼的位置、加粗、用较大的字号来突出体现等等，让招聘者能更容易看到。

  2. 无关紧要的内容不用写，简历不是比谁字数更多。一个比较简单的判断标准是：这个内容会让你看起来变得牛逼吗，如果一点也不会，那就删了吧。

  3. 尽量使用总结性的话语概括而不是冗长的描述，可以参考 STAR 法则。简单来说就是：遇到了什么问题、怎么解决的、带来了多少价值。

简历应该包含哪些模块

一份合格的简历通常应该有以下几个模块：基本信息、教育背景、专业技能、工作和项目经历、自我评价。如果以前有获得过一些荣誉，还有可以有个曾获荣誉。

基本信息

这一栏没啥好说的，放在简历第一栏，然后整齐、完整的写好必要的信息就行了。

必要的几项：姓名、性别、出生年月、电话、邮箱、现居住地、期望工作地、工作年限。

可以加的几项：

  1. 籍贯：说不定碰到老乡就放水了；
  2. 微信：万一电话打不通时，还可以通过微信联系你，邮箱实时性不强，所以微信还是比较方便的。

这一模块有一个容易有分歧的问题是：要不要放个人照片。

我自己是不放的，所以我的建议是不放。因为这一项并不是必须的，当一个选项不是必须的时候，你不放就不会给别人扣分的机会，如果你放了就给机会了。

当然，不排除有老哥觉得自己长的特别帅，hr又通常是妹子，可以尝试放一下，说不定就加分了。

教育背景

通常放在基本信息下面，每一段经历一行，也是整齐、完整的写好必要的信息就行了。

从大学的经历写起，通常包含：时间、学校、专业即可。如果成绩优秀的可以将平均学分绩点啥的加上。

有些人会罗列自己学过的学科，什么高等数学、离散数学，这些其实就不用了，基本不会有人关心这个。

专业技能

来了来了，简历里比较关键的一项。

这一栏的几个关键点是：

  1. 罗列出你会的技能 ，HashMap、MySQL、JVM、Redis 等等。

  2. 你对该技能的掌握程度：了解、熟悉、熟练（掌握）、精通。这边精通被我划掉了，这个词最好别用，使用这个词通常不会让人觉得你厉害，更多是觉得你无知。

  3. 技能不是随便罗列，要分组，例如：IO、集合、多线程、反射这些可以放一行，都是属于 Java 基础。

    常见的分组有：计算机基础、Java基础、SSM框架、JVM、SQL、分布式和中间件、设计模式等。

  4. 全部内容最好控制在十行或者以内，太长了就容易没有重点。

  5. 突出亮点。

    对于亮点不多的同学来说，可以写的更细一点，看过哪些源码，可以直接写上去，例如：深入学习过 HashMap、ArrayList、LinkedList 等常用集合的源码。

    对于经验丰富亮点较多的同学来说，可以写的总结一点，例如：对常用的 JDK 源码有深入的研究。

  6. 非本岗位的相关技能，例如很常见的前端相关知识：Html、js、ajax 等。我觉得如果内容实在不多的话，可以写一下，但是如果内容已经很多了，那就可以不写了。

工作(项目)经历

简历里另外一个比较关键的一项，工作经历和项目经历可以写一起，按时间倒序，按公司分组。

主要包含几个信息：工作时间、公司名称、项目名称、担任的角色、项目背景、主要职责。

  1. 工作时间：莫啥好说的，正常写即可。

  2. 公司名称：这边有个重点，有的同学比较老实，会写合同上的公司名称，例如：上海拉扎斯信息科技有限公司、北京三快在线科技有限公司、上海寻梦科技有限公司。

    这边其实没有必要哈，直接写公司更广为人知的名字即可，例如：饿了么、美团、拼多多，不然万一有招聘者不知道被刷就亏大了。

  3. 项目名称：也是正常写就ok。

  4. 担任的角色：如果只是普通的开发者，可以写核心开发，如果是主要负责，则可以写XX模块负责人、XX域负责人、项目负责人。

  5. 项目背景：比较关键的一项信息，项目背景要尽量简短，最好不超过2行，并且又要能让招聘者看出你究竟做了个什么东西，同时还要尽量看起来牛逼点。

    这个比较考验总结能力，可以多花点时间想想，然后可以让其他没有参与过该项目的同学模拟看一看是否能看得懂。

  6. 主要职责：本栏最重要的内容，内容可以是：做过的牛逼需求、解决过的复杂问题等等。关键词还是：尽量简短、突出重点、STAR 法则。

几个常见的样例如下，招聘者喜欢看的关键词我都加粗了：

样例1：参与XX项目从0到1的搭建，负责项目的架构设计、技术选型和整体环境搭建。

样例2：引入了XX技术/框架，解决了XX问题，提升了XX指标50%。

样例3：搭建XX工具，实现了XX功能的可视化、配置化、自动化（智能化）。

样例4：沉淀了XX技术，复用到XX个部门/小组，**提升了开发效率50%**。

样例5：负责系统整体的稳定性保障，使用隔离、熔断、降级、限流、XX，保障了系统可用性99.99%以上、XX指标99.99%以上。

样例6：负责X个人的小团队，团队同学在部门表现突出，XX指标排名第1，获得XX荣誉。

自我评价

自我评价主要要让招聘者更多的了解你这个人的优点，所以尽量写一些优秀的习惯、特质。

跟工作不是很相关的、比较日常的东西就不用写了，例如我经常看到有同学写：喜欢打羽毛球、喜欢打篮球等等，喵喵喵~

可以展现自己的工作态度，例如：工作积极主动、工作效率高、责任心强、注重细节、追求完美。

可以展现自己的学习能力，例如：学习能力强、解决问题能力强，喜欢钻研技术，喜欢学习源码，经常利用业余时间进行学习。

可以展现自己的思考和总结能力：能及时进行思考与总结，对相关知识进行沉淀与分享。

可以展现自己的团队精神：为人乐观大方，有很强的团队意识，乐于分享，乐于帮助别人，乐于与人沟通交流。

曾获荣誉

这个也没啥好说的，把自己获得过的一些荣誉都列一下。

可以是学生时期的，例如：优秀毕业生、优秀学生等等、ACM XX奖。

也可以是工作之后的，例如：X年优秀员工奖、X年难题攻关奖等等。

其他内容

技术博客、github：这个如果有不错的技术内容的话可以放，但是如果没啥东西就别放了。

实习经历：社招的话，我感觉就可以不放了，但是如果你是牛逼厂的实习的话可以放一下。

反正就一个原则，这个东西能让别人看了觉得你更牛逼了，那就放，否则，就不放。这条原则适用于整个简历。
