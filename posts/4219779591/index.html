java:-source 1.5中不支持diamond运算符（请使用-source 7 或更改版本以启用diamond运算符）

在项目的设置中选择Modules，每个项的Sources-Language level都改成7（默认是5）

Compilation failed: internal java compiler error

Settings--Build,Execution,Deployment--Compiler--Java Compiler：更改右侧Module的每个项为1.7（项目要求jdk 1.7。若要求1.8则要改成1.8）

Intellij IDEA junit 使用之org.junit不存在

File -> Project Struct… -> Libraies -> 点击绿色的加号 -> Java -> 找到 IDEA 安装路径下的 Lib 中的junit-4.12 -> 确定完就行了，点击OK就出去了

junit 单元测试报错：java.lang.NoClassDefFoundError: org/hamcrest/SelfDescribing

所以现在有两个办法解决：

  1. junit版本降到4.10

  2. 导入hamcrest-core-1.3.jar

    Project Structure–Libraries–”+”–找到D:\IntelliJ IDEA 2020.2.1\lib\hamcrest-core-1.3.jar

idea修改jsp不会热更新

mvn不是内部或外部命令，也不是可运行的程序或批处理文件

现象：配置了maven环境，在cmd内输入mvn -v正常；在idea上显示“mvn 不是内部或外部命令，也不是可运行的程序或批处理文件”

解决：idea“以管理员身份运行”打开

idea导入时有很多错

  1. mvn clean install 得到的jar包是最新的，除非用其他方式修改jar包中的内容而不修改源代码

  2. 平时用mvn install，不进行clean来节省时间，保险还是用mvn clean install生成最新的jar包或其他包

  3. 不想用mvn clean又想保证jar包最新，建议加 -Djar.forceCreation参数，如mvn install -Djar.forceCreation

idea卡在parsing java

网上说是jdk版本太高

  1. File–Settings–Build，Execution，Deployment–Compiler–Java Compiler：更改右侧的Target bytecode version

  2. Project Structure–Project：SDK版本；Project language Level

    Project Structure–Modules：Language Level

  3. 调整VM option idea64.exe.vmoptions

  1. idea：File–Settings–Build，Execution，Deployment–Compiler

    调整Build process heap size(Mbytes)，调整为2048

无效的源发行版：16

解决：project Structure -> Modules -> 选择项目 -> 点击sources,更改 Language Level为对应的JDK版本

Kotlin not configured

原文

  1. 选中项目按F4，打开Project Structure（或者File》Project Structure）

  2. Project内确保Project SDK为java jdk，注意设置 Project complier output 目录

  3. 打开Modules，添加Kotlin右侧勾选KotlinJavaRuntime

  4. 打开Modules选中Kotlin，右侧的Target platform选为JVM1.8
