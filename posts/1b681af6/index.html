Python-函数

Posted on 2022-05-30 | In Python

函数一

概述

  • 内涵

    • 对“程序逻辑”进行

      • 结构化和过程化

    • 面向过程的程序设计

      • 基本单元

  • 定义

    • def

    • return

      • 随时返回

        • 函数结果

      • 返回形式

        • 无“返回值”

          • 方法

            • 不用return

            • 仅return

              • 不加返回值

          • 返回对象类型

            • None

        • 一个值/对象

          • ps

            • 若返回多个对象

            • 本质上以“元组”返回

      • 执行到return时

        • 停止执行函数内余下的语句

    • 空函数

      • pass
        • 占位符
          • 让代码先运行起来

  • 注意

    • vs

      • 函数的引用

        • 访问

        • 别名变量

          • 函数名

            • 本质上即“变量”

          • 可有多个别名

            • 对同一“函数对象”的引用

      • 函数的调用

      • e.g

        • def foo(): …

          • 新建函数对象

          • 赋值给foo

        • bar = foo

          • bar引用同一函数对象

        • bar()

          • 同foo()
            • 调用函数

    • 函数

      • 可作为“参数”
        • 在函数体内被调用

装饰器

Decorator

  • 在代码运行期间，给函数，动态增加功能

  • 本质上

    • 一个“返回函数”的高阶函数

    • 内涵

      • 输入参数

        • A函数

      • 返回值

        • A函数

  • 形式

    • @decorator(dec_opt_args) def func2Bdecorated(func_opt_args): …

  • 应用

    • 引入日志

    • 增加计时逻辑

      • 检测性能

多个装饰器“堆叠”

  • @deco2(deco_args) @deco1 def func()
    • 等价于
      • def func(): func = deco2(deco_args)(deco1(func()))

e.g

  • 定义装饰器

    • def log(func): def wrapper(*args, **kw): print(&apos;call %s():&apos; %func.name) return func(*args, **kw) return wrapper

      • log

        • 装饰器函数

      • 参数

        • *args, **kw

        • 可以接受任意参数的调用

  • 使用装饰器

    • @log def now(): print(&apos;2020-07-24&apos;)

      • @log放到now()函数的定义处

      • 相当于执行了语句 now=log(now)

参数

完整语法

  • func(positional_args, keyword_args, *tuple_nonkw_args, **dict_kw_args)

  • 参数定义的顺序

    • 按传递方式

      • 位置–关键字–包裹位置–包裹关键字

    • 按参数类型

      • 必选–默认–可变–关键字

参数传递方式

  • 位置传递

    • 依次传入参数
      • 位置参数/必选参数

  • 关键字传递

    • 声明了“默认值”的参数

      • 默认参数

    • 方式

      • 参数的名字

        • 提供，对应名字传入

        • 不提供，按顺序依次传入

    • 参数默认值

      • 必须指向“不变对象”

        • 如果需要用到list作为参数

        • 不要直接用

        • 改用None

          • if L is None: L = []

  • 包裹传递（可变参数）

    • 目的

      • 传入“不定个数”的参数组

        • 元组

          • 非关键字参数

        • 字典

          • 关键字参数

    • 方式

      • 包裹位置参数

        • 非关键字可变长参数（元组）

        • 定义函数

          • def func(*tuple_name)

            • 封装成元组

            • 任意数量的位置参数

          • tuple_name

            • 收集所有的参数

            • 根据位置

              • 合并成一个元组（tuple）

        • 使用函数

          • 先组装 list/tuple 再传入

            • list/tuple 前加*号

            • e.g

              • a=[1,2,3] fun_name(*a)

          • 直接传入一串值

            • e.g
              • func(1,2,3)

      • 包裹关键字参数

        • 关键字参数（字典）

        • 定义函数

          • def func(**dict_name)

            • 任意数量

              • 封装成“字典”

            • 允许参数

              • 缺失

              • 不按顺序

        • 使用函数

          • 先组装 dict 再传入

            • dict 前加**号

            • e.g

              • a = {key1:value1, key2:value2} func_name(**a)

          • 直接传入多个name=&apos;value&apos;

            • e.g
              • func(a=1, b=2)

    • 解包裹

      • 目的

        • 形如 func(*args, **kw)

          • 首先拆解args

          • 按顺序传给

            • 必选参数–默认参数–可变参数（如果还有值剩余）

          • 再拆解 kw

          • 传给

            • 关键字参数

        • *, **

          • 用来拆解 list/tuple & dict
            • 单个参数

      • 方法

        • func(*args)

          • tuple的每一个元素，对应一个“位置参数”

        • func(**dict)

          • 词典的每个键值对，作为一个“关键字参数”

      • vs

        • 包裹

          • 定义函数时用

        • 解包裹

          • 调用函数时用

处理“参数传递”的两种形式

  • 值传递

    • 参数

      • 基本s数据类型

    • 内涵

      • 变量传递给函数后

        • 函数在内存中

          • 复制一个新变量

          • 不影响原有变量

      • 指向新的引用对象

  • 指针传递

    • 参数

      • list

    • 内涵

      • 变量传递给函数的是指针

        • 指针指向

          • 序列在内存中的位置

        • 在函数中对list的操作

          • 在原有内存中进行

          • 从而影响原有变量

      • 原引用对象被改变

  • 可以先检查“参数类型”

    • isinstance()

函数二

函数式编程

  • 释义

    • 一种“编程范式”

      • 如何编写程序的方法论

        • e.g

          • 面向对象

          • 面向过程

    • 属于“结构化编程”

      • 采用

        • 子程序

        • 程式码区块

        • for / while循环

  • 思想

    • 把运算过程
      • 尽量写成
        • 一系列嵌套的函数调用

  • 特点

    • 允许

      • 把函数作为参数

        • 传入另一个函数

      • 返回一个函数

  • Python

    • 并非“函数式编程语言”

    • 但支持一些

      • 函数式编程语言的构建

        • 匿名函数

        • BIF

          • filter, map

        • 偏函数

    • 本质上

      • 通过“封装对象”实现“函数式编程”

匿名函数

  • 关键字

    • lambda
      • 不需要以
        • 标准的方式
          • 来s声明

  • 表达式

    • lambda[arg1,[, arg2, …, argN]]: expression

    • 同一行

      • 定义体 + 声明

      • 不用写return

  • 使用

    • 方法一

      • 赋值给一个变量

        • func=lambda x,y: x+y

        • 内涵

          • lambda生成一个函数对象

          • 参数为 x, y

          • 返回值为 x+y

          • 函数对象赋值给func

      • 用变量来调用该函数

        • func(3,4)
          • func的调用与正常函数无异

    • 方法二

      • 作为返回值返回
        • def build(x, y): return lambda: x * x + y * y

高阶函数BIF

  • 定义

    • 一个函数接收“另一个函数”作为“参数&quot;

    • 可与lambda完美结合

  • apply()

    • apply(func[, nkw][, kw])

    • 已淘汰

      • 已经允许函数用”可变参数“

      • 没有存在的价值了

  • filter()

    • filter(func, seq)

      • 将函数对象

        • 依次作用于每一个元素

      • 根据返回值是 True/False

        • 决定保留/丢弃该元素

    • e.g

      • filter(lambda n: n%2, allNums)

      • 可用”列表解析“实现同样功能

        • [n for n in allNum if n % 2]

  • map()

    • map(func, seq1[, seq2, …])

      • 将函数对象

        • 依次作用于每一个元素

      • 每次作用的结果

        • 储存于返回的list中

      • 可用有n个列表

        • 对应函数的n个参数

    • e.g

      • map(lambda x: x**2, range(5))

        • 等价于
          • [x**2 for x in range(5)]

      • map((lambda x, y: x + y), [1,2,3], [6,7,9])

        • [7, 9, 12]

      • map(None, [1,3,5], [2,4,6])

        • [(1,2), (3,4), (5,6)]

        • 思想类似

          • zip()

  • vs

    • reduce()

      • from functools import reduce

      • reduce(func, seq[, init])

        • 累进作用

        • func

          • 二元函数

          • 接收两个参数

            • 上一次

              • 返回值

            • 序列下一个元素

        • init

          • 初始化器

          • 若设定

            • 第一次计算为
              • 初始化q器 & 第一个序列元素

      • e.g

        • reduce(lambda x,y: x+y, range(5))

          • 相当于
            • (((0+1)+2)+3) + 4

        • reduce(func, [1,2,3]) = func(func(1,2), 3)

偏函数

  • 概述

    • 偏函数应用（PFA）

      • Partial function application

    • 结合了

      • 函数式编程

      • 默认参数

      • 可变参数

    • 使用场景

      • 函数的参数太多

        • 需要j简化

      • 创建一个新函数

        • 固定住
          • 原函数的部分参数

  • 应用

    • 模块

      • functools

    • 创建

      • from functools import partial partial(func, *args, **keywords)

    • e.g

      • 固定“位置参数”

        • add1 = partial(add, 1)

          • # add1(x) == add(1, x)

        • int_new = partial(int, 2)

          • # int_new == int(2, base)

      • 重设“关键字参数”

        • int 2 = partial(int, base=2)

        • int2(&apos;10010&apos;)

          • 18

        • int2(&apos;10010&apos;, base=10)

          • 可g更改，仍为关键字参数

  • 变量作用域

    • 标识符的作用域

      • 其声明
        • 在程序里的可应用范围

    • 全局变量

      • 函数内

        • 可以引用

        • 不能修改

          • 除非声明global

      • 除非被删除

        • 否则存活到
          • 脚本运行结束

    • 局部变量

      • 在函数内定义

        • 只允许函数内访问

      • 存在与否

        • 依赖于
          • 函数是否处于“活动状态”

    • 变量搜索顺序

      • 局部作用域 –&gt; 全局域

      • ps

        • 局部变量
          • 可能隐藏 / 覆盖
            • 全局变量

  • 返回（回调）函数

    • 函数作为“结果值”返回

      • 每次调用

        • 都会返回

          • 一个新的函数

        • 即使传入

          • 相同的参数

        • 调用结果

          • 互不影响

    • “闭包”的程序结构

      • 内部函数

        • 引用
          • 外部函数的“参数和局部变量”

      • 调用外部函数

        • 返回
          • 已保存“参数和变量”的内部函数

    • 牢记

      • 返回”闭包“时

      • 返回函数不要引用

        • 任何”循环变量“

          • 如果一定要引用

            • 再创建一个函数

            • 用该函数的参数

            • 绑定循环变量当前的值

        • or

          • 后续会发生变化的变量

  • 递归函数

    • 函数在内部调用”自身“

    • 注意

      • 防止”栈溢出“

        • 原因

          • 每进入一个函数调用

            • 栈就会加一层栈帧

          • 每当函数返回

            • 栈就会减一层栈帧

          • 递归调用的次数过多

            • 导致z栈溢出

        • 解决

          • 尾递归优化

            • 本质上

              • 效果=循环

            • 函数返回时

              • 调用自身本身

            • return

              • 不能包含表达式

          • 缺陷

            • 若没有

              • 针对尾递归继续做优化

            • 任何递归函数

              • 都存在”栈溢出“的问题
