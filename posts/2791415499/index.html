Java字节码

发表于 2021-03-09 更新于 2021-03-31 分类于 java
本文字数： 2.3k 阅读时长 ≈ 2 分钟

字节码

  • 解释执行

源码 –> Parser –> 抽象语法树（AST）–> 解释执行

  • 编译执行

源码 –> 编译&链接 –> 机器指令 –> 在硬件上直接执行

  • 字节码

源码 –> 编译 –> 字节码 –> 在虚拟机上执行

因为解释执行弱点：解释器需要遍历抽象语法树来解释执行，效率非常低，被字节码打败了。

又因为编译执行占据了系统级编程的地盘，操作系统、网络、数据库、虚拟机等核心领域都是。所以无法被字节码替代。
