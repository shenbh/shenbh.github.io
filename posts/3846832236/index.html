监控值的获取方法–冷启动

  • 启动 App 命令

adb shell am start -W -n package/activity

停止 App 命令

adb shell am force-stop package

找到包名的操作步骤：

  1. adb logcat | grep START监控指令（mac系统上的是grep；windows上的要换成findstr）
  2. 手动打开App，log 会打印出新打开的 App 的信息
  3. 获取正在运行的app的包名adb shell am monitor
  4. 获取自己安装的app的包名 adb shell pm list packages -3

监控值的获取方法–热启动

  • 启动 App 命令

adb shell am start -W -n package/activity

  • 停止 App 命令

adb shell input keyevent 3 相当于点击了系统的 back 键

CPU

  • 获取数据

adb shell dumpsys cpuinfo | grep packagename

流量

  • 获取进程 ID 流量

adb shell cat /proc/pid/net/dev

  • 获取运行 app 的进程 ID

adb shell ps | grep packagename

电量

  • 获取电量 （关注 level值）

adb shell dumpsys battery

  • 切换非充电状态

adb shell dumpsys battery set status 1 //2是充电状态

内存

  • 获取内存

adb shell top

VSS - Virtual Set Size 虚拟耗用内存

RSS - Resident Set Size 实际使用物理内存

adb shell top -d 1 //其中-d 表示刷新频率

adb shell top -d 1 > meminfo

cat meminfo | grep com.android.browser

  • 设置断点的方式

    1. 通过工具栏设置断点功能

    设置断点：bpu 指定站点（如：bpu www.google.com）

    清除断点：bpu 回车

Fiddler 设置断点修改 Response

  • 设置断点的三种方式

    1. 通过工具栏设置断点

    设置断点：Rules->Automatic Breadpoints->After Response

    清除断点：Rules->Automatic BreadPoints->Disabled

    1. 通过命令设置断点

    设置断点：bpafter www.google.com

    清除断点：bpafter 回车

API 接口测试时的问题

  1. 后端服务提测时间早于客户端

  3. Fiddler 的 composer 无法实现自动化验证

Postman

  1. GET、POST 的请求都有个 Tests，里面可以添加验证请求成功的规则。

  2. 可以把多个请求放到同一个文件夹内，然后运行文件夹进行多个接口一起验证（还可以设置请求次数和间隔）

自动化工具简介

Appium的介绍

是个开源、跨平台的自动化测试工具。用于测试Native（原生）和Hybrid（混合）应用。支持IOS、Android和FirefoxOS平台。

  • Android平台

    基于UIAutomator框架

理念

  • 无需重新编译应用
  • 不局限于语言和框架
  • 无需重复造轮子，接口统一
  • 无论精神上，还是名义上，必须开源

特点

  • 跨架构，Native、Hybrid、Webview
  • 跨设备，Android、IOS、Firefox OS
  • 跨语言，Java、Python、Ruby、PHP、JavaScript
  • 跨进程，不依赖源码（基于UIAutomator）

架构

UiAutomator controller：把Bootstrap.jar（通过adb命令）从PC端推送到手机端；

Bootstrap.jar监听端口号

UiAutomator command server（是个TCP server）：用来接收appium（UiAutomator command client）发送来的各种指令；会启动Bootstrap.jar；用来控制App
