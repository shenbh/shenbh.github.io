新增项目步骤

  1. 登录服务器端https://gitlab.51zcm.cc/
    new project 在自己用户名路径下新建工程，命名 workreport

  2. 本地新增文件夹xxx，进去xxx文件夹，右击git bash here

    $ git clone http://gitlab.51zcm.cc/自己用户名/workreport.git

  3. 在xxx文件夹里面的workreport里面拷贝放置《工作汇报.xlsx》

  4. 初始化
    $ git init

    添加远程仓库地址
    $ git remote add origin http://gitlab.51zcm.cc/自己用户名/workreport.git

添加本地代码
$ git add *

提交本地代码
git commit -m "新建"

提交到远程仓库?
$ git push origin master

同一份代码备份到不同的git（gitee和github）

Windows下配置git

  • https://git-for-windows.github.io/ 下载并安装git

  • 随便找个目录，右键打开Git Bash Here，设置（创建/修改）git的user name和email

  • 配置ssh key

  • 打开C:/User/用户/shenbh/.ssh/rsa.pub，复制其中的内容，黏贴到gitee、github的公钥配置中

fatal: HttpRequestException encountered

从 https://github.com/Microsoft/Git-Credential-Manager-for-Windows/releases/tag/v1.14.0 下载GCMW-1.14.0.exe并安装

Windows10上git配置github和gitee

github项目迁移到gitee

  1. github上点clone，复制HTTPS链接

  2. 新建仓库：在gitee上“新建仓库”（不用写名称什么的）

    导入已有仓库：滑到底部，点击“导入已有仓库”，黏贴第1步中复制的链接）

    创建：点击创建，等待即可

SourceTree

合并分支到master

  1. 双击选中master
  2. 在分支（“开发”）上右键，选择“合并开发至当前分支”

问题

fatal: HttpRequestException encountered解决方法

push到远程仓库的时候会弹窗让输入账密，输完后报上面的错误。

解决：从网址：https://github.com/Microsoft/Git-Credential-Manager-for-Windows/releases/tag/v1.14.0 下载 GCMW-1.14.0.exe 并安装即可。

GIT报错：connect to host github.com port 22: Connection timed out

在.ssh文件夹下新建一个 config 文件（不要后缀名），填入以下内容

Host github.com /服务器地址为github地址/
User “xxx@xx.com“ /github上的注册邮箱 为用户账号/
Hostname ssh.github.com /服务器地址为github地址/
PreferredAuthentications publickey /采用公匙/
IdentityFile ~/.ssh/id_rsa /公匙文件路径/
Port 443 /修改端口为443/
