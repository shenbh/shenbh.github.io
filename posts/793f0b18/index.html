git commit -m "新建"

提交到远程仓库?
$ git push origin master

5个提高效率的命令

stash

临时保存更改的，然后切分支

场景：正在开发的内容临时保存（但不想要有提交记录），此时要切分支更改bug。

解决：

  1. 可以用 git stash save "备注的内容"来保存在本地
  2. 恢复代码用 git stash apply

其他完整命令：

  1. 当有多条stash，可以指定操作stash，先用stash list列出所有记录

    想要第二条记录：

    pop，drop同理

reset –soft

回退已提交的commit，并将commit修改的内容放回到暂存区

git reset --hard 回退到某一个节点

git reset --soft 回退到某一个节点，同时保留节点的修改内容

以上针对的未push的commit。对于已经push的commit也可以使用该命令，不过下次提交（由于远程分支和本地分支有差异）需要强制推送git push -f

注意：reset –soft指定commit号时，会将该commit到最近一次commit的所有修改内容全部恢复。

问题

GIT报错：connect to host github.com port 22: Connection timed out

在.ssh文件夹下新建一个 config 文件（不要后缀名），填入以下内容

Host github.com /服务器地址为github地址/
User “xxx@xx.com“ /github上的注册邮箱 为用户账号/
Hostname ssh.github.com /服务器地址为github地址/
PreferredAuthentications publickey /采用公匙/
IdentityFile ~/.ssh/id_rsa /公匙文件路径/
Port 443 /修改端口为443/
