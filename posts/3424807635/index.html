Fiddler抓取Chrome的Https

  1. 清除电脑上的fiddler证书
    1. C:\Users\Administrator\AppData\Roaming\Microsoft\Crypto\RSA 这个目录下的都删除
    2. WIN+R，输入 certmgr.msc，弹窗中>操作>查找证书，输入 fiddler，删除查到的证书
    3. 清除浏览器上的证书文件，chrome上输入 chrome://settings/，隐私设置和安全性》安全》管理证书，弹窗切换到页签“受信任的根证书颁发机构”，找到 FiddlerRoot 字样的证书并删除
  2. 打开Fiddler，Tools》Option，切换HTTPS页签，勾选 “Capture HTTPS CONNECTs”、“Decrypt HTTPS traffic”、“Ignore server certificate errors(unsafe)”，点击Action进行生效。
    ps：记住要先推出杀毒软件
    最新版Fiddler的官网下载地址：https://www.telerik.com/download/fiddler
  3. chrome上输入 chrome://settings/，隐私设置和安全性》安全》管理证书，弹窗切换到页签“受信任的根证书颁发机构”，点击“导入”进行导入证书
  4. 打开Fiddler即可抓包了，如果抓取的包是错误的，需要把Autoresponder–enable rules这一行的几个选项反勾选掉。

Fiddler抓取手机的Https

  1. 安装Fiddler插件

    下载插件https://www.telerik.com/fiddler/add-ons，找到“CertMaker for iOS and Android”这一项进行下载【双击安装插件，可能需要下载更新的fiddler版本】

  2. 配置Fiddler代理

    tools->options，切换到HTTPS页面

    然后在弹出的弹窗中选择yes、确定

    出现“Added Fiddler’s root certificate to the Machine Root List.”就说明证书安装成功了。

  3. 重启fiddler。

安卓手机

  1. 配置手机的证书

    打开手机浏览器输入电脑的ip和端口（fiddler》tools》options》Connections中配置的端口），如我的192.168.1.111:8888，然后点浏览器中的证书链接进行下载并安装。

    直接在浏览器上打开这个下载的文件，就会提示安装

  2. 配置手机的代理（手机和电脑在同一个网络中）

Fiddler模拟接口响应数据

方法一 ：手动修改

  1. 设置断点：Rules》Automatic Breakpoints》After Responses

  2. 重新请求数据，在右下方框中找到“TextView”修改底下的内容，再按Run或者“Run to Completion”

方法二：把结果提前改好保存成文件，让对应的请求去加载这个文件【不用设置断点】

将右下方“TextView”获取到的内容保存在.html文件内，将修改后的.html文件加载到“AutoResponse”窗口“Rule Editor”

  1. 获取接口返回数据：右下方“TextView”获取到的数据保存在.html文件内（看右下角的“View in Notepad”）

  2. Fiddler设置静态接数据：

    点击模拟接口，然后切换窗口到“AutoResponse”窗口，勾选两个勾选框“Enable rules”、“Unmatched requests passthrough”，点击Add Rule，这样就把此接口添加到了Rule里。

  3. 把刚才获取的数据文件xx.html加载到此处（需要提前修改.html文件内容）

    上一栏选择框，删掉前面的“EXACT：”
