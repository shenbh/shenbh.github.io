SQL相关

Posted on 2020-01-07 | In 数据库

概念

主键

PRIMARY KEY

主键：一张表中，用于唯一标识一条记录的字段组（属性组）

外键

FOREIGN KEY（在子表中）

外键：一个表A的主键在另一个表B中。那么B中这个字段可以作为A表的外键。（存在与外面的主键就是外键）

主键是唯一的，用于标识一张表。外键可有多个，用于建立表和表的关系。

为t2插入数据，由于外键约束的存在，t2(t_id)列的数据必须都来自于t1(id)列，如果t1(t_id)不存在的数据，插入执行会报错

t2的删除操作不受外键的约束

对t1执行删除/更新操作，如果删除/更新的主键值在子表t2中出现，那么就删除/更新失败

外键定制三种约束模式：

  • restrict和no action：父表更新或删除时，子表有匹配记录，则禁止父表的更新和删除。（默认选项）
  • cascade：父表更新或删除时，子表有匹配记录，则父表操作成功，同时更新或删除子表匹配项。
  • set null：父表更新或删除时，子表有匹配记录，则父表操作成功，同时子表的匹配项设置为null值（前提能设置为null）

逻辑操作符

  • IS NULL(是否为空)（= NULL 是否含有NULL字符）

  • BETWEEN（在某两个值之间）

  • IN（一系列值中的值）

  • WHERE salary IN （200,300,400）工资必须是200,300,400之中的值

  • LIKE（相似值得比较）

  • 使用匹配符将一个值同其相似的值比较

  • 匹配符包括：-代表一个字符；%代表多个字符

  • 如：WHERE salary LIKE ‘-00’查找‘00’作为后两位数的值

  • EXITS（是否存在符合条件的数据）

  • 查询在指定表中是否存在一行符合某种条件的数据

  • UNIQUE（是否唯一）

  • 在特定的表中搜索每一行是否唯一

  • 测试满足条件的工资是否有重复的数据

  • ALL/ANY（一组数据的所有/其中的任何一个）

  • 与某个数据集中所有数据/任何一个进行比较

  • AND/OR（逻辑与/或）

    AND连接的条件必须都满足

    OR连接的条件至少满足一个

库函数

函数通常是列名或表达式相连系的命令

统计函数（累加、总和、平均值）

count：主要的统计函数

COUNT:统计行数或不为NULL的列数

语法:COUNT[(*)|(DISTINCT|ALL)]COLUMN NAME)

注意：DISTINCT 统计唯一的行；DISTINCT不用用于COUNT(*)，只能用于COUNT（colume_name）

举例：
SELECT COUNT(\*) FROM EMPLOYEE_TBL
SELECT COUNT(EMP_ID) FROM EMPLOYEE_TBL
SELECT COUNT(DISTINCT SALARY)FROM EMPLOYEE_PAY_TBL

SUM:求和

语法:SUM([DISTINCT]COLUMN NAME)

举例：
SELECT SUM(SALARY) FROM EMPLOYEE_PAY_TBL
SELECT SUM(DISTINCT SALARY)FROM EMPLOYEE_PAY_TBL

MAX:求最大值

MIN:求最小值

语法:
MAX([DISTINCT]COLUMN NAME)
MIN([DISTINCT]COLUMN NAME)

AVG:求平均值

语法
AVG([DISTINCT]COLUMN NAME)

举例：
SELECT AVG(SALARY)FROM EMPLOYEE_PAY_TBL
SELECT AVG(DISTINCT SALARY)FROM EMPLOYEE_PAY_TBL 计算唯一的工资的平均值

LOWER(str)：字符串转小写

高级用法

API

Join

Join

join 用于根据两个或多个表中的列之间的关系，从这些表中查询数据。

如果表中有至少一个匹配，则返回行

Inner Join

在表中存在至少一个匹配时，INNER JOIN 关键字返回行。

注释：INNER JOIN 与 JOIN 是相同的。

左连接 left join … on

LEFT JOIN 关键字会从左表 (table_name1) 那里返回所有的行，即使在右表 (table_name2) 中没有匹配的行。

右连接 right join … on

RIGHT JOIN 关键字会右表 (table_name2) 那里返回所有的行，即使在左表 (table_name1) 中没有匹配的行。

自连接

Full Join

只要其中一个表中存在匹配，就返回行

UNION拼接

UNION

UNION 操作符用于合并两个或多个 SELECT 语句的结果集。

UNION 结果集中的列名总是等于 UNION 中第一个 SELECT 语句中的列名

注释：默认地，UNION 操作符选取不同的值。如果允许重复的值，请使用 UNION ALL。
