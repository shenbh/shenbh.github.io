Eclipse配置Java后台环境

Posted on 2020-07-17 | In 工具

Eclipse导入项目

准备环境：

  1. maven，配置环境

  2. Eclipse

  3. jdk7

    并配置环境变量

步骤：

  1. Eclipse中配置maven

maven的setting中增加

配置本地仓库路径

配置项目jdk版本

  1. 导入项目

Import–>Existing Maven Projects

  1. 项目不报错的情况下开始，运行

-Xms512m -Xmx1380m -XX:PermSize=128M -XX:MaxPermSize=256m

点击Debug即可，需要等待（2~3min）

启动成功标识：

浏览器请求验证

又一城eclipse导入项目

eclipse导入项目前配置

  1. 新建文件夹当作workspace
  2. 打开eclipse，选择编码为utf-8（Windows->Preferences->General->Workspace选择Text file encoding为UTF-8）

  1. 设置maven（Windows->Preferences->Maven->Installations->Add添加本地maven地址）

eclipse导入项目并运行

用eclipse的svn导入几个项目到本地

删除eclipse内的项目，再用maven重新载入

先修改U1Shop.Shop/pop.xml，打开所有的依赖注释

删除eclipse内所有项目的引用（记得别勾选删除本地文件）

用maven方式重新导入本地的项目

去除Seller、H5等项目的JS校验

去掉全局关于xml、html、js的自动校验

更改每个项目的jdk版本为1.7

定制项目TmallShopConstant中要更改ENVIRONMENT（本地运行的话改成0）

PMS代码中要放开这个注释（SecurityContextHolder.setStrategyName(SecurityContextHolder.MODE_GLOBAL);）

运行前检查下数据库、redis的连接

jetty启动

选中对应要启动的代码，编辑Debug Configurations

配置VM所需要的大小，防止默认的过小导致启动超时

要调试的话需要关联相应的模块，否则代码跳不过去

启动成功

访问路径：localhost:端口或者本地ip+端口

注意：H5访问是 http://localhost:端口/home?tmallshopid=15886

（其中localhost也可以换成本地ip）

tomacat启动

配置tomcat环境变量

校验：

  1. 在命令行窗口输入 startup
  2. 在浏览器输入localhost:8080

eclipse设置Tomcat和启动

添加全局tomcat的运行环境

eclipse设置打开eclipse选择顶部Windows–>Preferences–>server–>Runtime Environments点击Add选择自己对应的Tomcat版本（以及正确的路径）、关联jre

配置启动项目的参数
添加Servers

Windows-->Show View-->Other搜索Server点击添加Servers

新建Server（Tomcat）

控制台切换到Servers右键–>new–>server选择对应的Tomcat版本将所要启动的服务加载进去

更改tomcat配置

新建好服务之后双击打开刚刚创建的Tomcat服务

更改Overview

在Overview需要修改的地方：

  1. General Information-Server name：运行服务的名称

  2. General Information-Open launch configuration-Arguments-VM arguments：最后添加一行

    用来修改VM的大小

  3. Timeouts-Start：修改启动的超时时间

  4. Ports-HTTP/1.1：修改服务的端口号

更改Modules

在Modules需要修改的地方

最后记得保存

启动tomcat服务

控制台-点开SjSupport-选择U1Shop.SjSupport-右键选择Debug启动

tomcat启动成功图

java.lang.ClassNotFoundException:net.sf.json.JSONException

用jetty的时候有时候会自己忽略一些jar，需要手动关联上

勾选上json-lib-2.4-jdk15.jar

出错 “严重: PWC6117”

查了网上说的是

但是我代码里没有这个问题。

原因：通过断点发现连接的数据库不对，而代码对少了个error_login的错误码的登陆失败情况的处理

解决：更改数据库连接；增加error_login错误码的处理。

No mapping found for HTTP request with URI [/] in DispatcherServlet with name ‘appServlet’

源码路径错误

A cycle was detected in the build path of project ‘EasyH5’….

解决：

  1. Eclipse/STS -> Window -> Preferences… -> Java -> Compiler -> Building -> Building path problems -> Circular dependencies -> 将Error改成Warning
  2. 或者查看工程之间是否真的需要循环引用，不需要的话去掉。

ASN1InputStream cannot be resolved to a type…

这是项目中需要手动添加jar依赖
